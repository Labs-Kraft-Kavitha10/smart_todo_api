# Smart Todo API

A small text-classification API that predicts the priority of a free-form
to-do item (`High`, `Medium`, `Low`) using a TF-IDF + Multinomial Naive Bayes
pipeline served by Flask. The repo also contains K6 performance tests
(ramp-up + spike) and a self-healing API test that tolerates field-name drift
in the response shape.

---

## Prerequisites

- **Python 3.10+** (developed on 3.14, but anything ≥ 3.10 should work)
- **K6 v2.x** for the performance tests. On macOS:
  ```bash
  brew install k6
  ```
  Other platforms: https://k6.io/docs/get-started/installation/

---

## Setup

```bash
git clone <repo-url> smart_todo_api
cd smart_todo_api
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Activate the venv each time you open a new shell with `source .venv/bin/activate`.

> **Shortcut:** if you have GNU Make installed (ships with macOS) the
> commands below all have one-word aliases. Run `make help` to see them,
> or jump straight to [Makefile shortcuts](#makefile-shortcuts).

## Train the model

```bash
python train.py
```

Reads `data/tasks.csv`, does an 80/20 stratified split, fits the pipeline,
prints accuracy / confusion matrix / classification report, and writes
`model.joblib` to the project root. Acceptance threshold is **≥ 70 %**
accuracy; the current dataset reproduces at **~93 %** with `random_state=42`.

---

## Start the API

```bash
python app.py
```

Flask binds to `0.0.0.0:5001`.

> **Note:** Port **5001** (not 5000). macOS reserves port 5000 for the
> AirPlay Receiver service by default. If you'd rather use 5000, disable
> AirPlay Receiver in *System Settings → General → AirDrop & Handoff* and
> change the `port=5001` line at the bottom of `app.py`.

### Endpoints

| Method | Path | Body | Response |
|---|---|---|---|
| `GET`  | `/health`  | — | `{"status": "ok"}` (200) |
| `POST` | `/predict` | `{"task": "..."}` | `{"priority": "...", "confidence": 0.xx}` (200) or `{"error": "Missing field: task"}` (400) |

Smoke test:

```bash
curl http://localhost:5001/health

curl -X POST -H "Content-Type: application/json" \
     -d '{"task":"Fix login bug before tomorrow demo"}' \
     http://localhost:5001/predict
```

## Run the K6 performance tests

Start the API in one terminal, then in another:

```bash
mkdir -p results

# Ramp-up: 0 → 10 → 50 VUs, ~4 min total
k6 run --out json=results/ramp_up_results.json k6/ramp_up_test.js

# Spike: 5 VUs → burst to 100 → back to 5, ~1m 25s
k6 run --out json=results/spike_results.json k6/spike_test.js
```

Each script writes an HTML report to the project root via K6's
`handleSummary` hook:

```bash
open k6_report_rampup.html k6_report_spike.html
```

Written analysis of the runs is in **`k6_analysis.md`**.

## Run the self-healing test

With the API running:

```bash
python self_healing_test.py
```

Exits **0** if the live response matches `expected_shape.json` (or a known
alias resolves drift), **1** otherwise. The design is explained in
**`healing_explanation.md`** and a 5-stage demonstration log is in
**`demo_log.txt`**.

To reproduce the demo manually:

1. Run the test against the original API — passes.
2. In `app.py`, change `"priority"` to `"level"` in the `/predict` response
   and restart — re-run the test: a `SCHEMA_DRIFT WARNING` is logged but
   tests still pass (alias hit).
3. Change `"level"` to `"rank2"` and restart — re-run: `SCHEMA_BREAK ERROR`,
   exit code **1**.
4. Restore `"priority"` — re-run: clean pass.

## Makefile shortcuts

If you have GNU Make available, every command above has a short alias:

| Command | What it does |
|---|---|
| `make help`      | List all targets |
| `make setup`     | `python3 -m venv .venv` + `pip install -r requirements.txt` |
| `make train`     | `python train.py` |
| `make run`       | `python app.py` (blocks; Ctrl+C to stop) |
| `make test-api`  | Three quick curl checks against `:5001` |
| `make k6-rampup` | K6 ramp-up test |
| `make k6-spike`  | K6 spike test |
| `make k6-all`    | Both K6 tests sequentially |
| `make heal`      | `python self_healing_test.py` |
| `make clean`     | Remove `model.joblib`, generated reports, `results/`, `__pycache__/` |
| `make clean-all` | Also remove the `.venv` |

Typical demo flow:

```bash
make setup       # one-time
make train       # one-time
make run         # leave running
# in another terminal:
make test-api
make k6-all
make heal
```

## End-to-end reproduction (TL;DR)

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python train.py                                       # build model.joblib
python app.py &                                       # start API on :5001
sleep 2
curl -X POST -H "Content-Type: application/json" \
     -d '{"task":"Patch security vulnerability"}' \
     http://localhost:5001/predict                    # smoke test

mkdir -p results
k6 run --out json=results/ramp_up_results.json k6/ramp_up_test.js
k6 run --out json=results/spike_results.json   k6/spike_test.js

python self_healing_test.py                           # exit 0 on success
```
